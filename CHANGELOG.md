## [v0.4.0] - 2026-01-25

### Highlights

- Schema codegen pipeline: Go types are now the source of truth
- New deployment platforms: Gemini CLI, ADK Go, CrewAI, AutoGen
- TypeScript SDK auto-generated from JSON Schema

### Added

- **Codegen Pipeline**: Go → JSON Schema → TypeScript/Zod
  - `tools/generate/` - Go schema generator using invopop/jsonschema
  - `sdk/typescript/scripts/generate-schemas.mjs` - TypeScript/Zod generator
  - `Makefile` - Orchestrates full codegen pipeline (`make all`)
- **New Platforms**: GeminiCLI, ADKGo, CrewAI, AutoGen with typed configs
- **Runtime Config**: RuntimeConfig, StepRuntime, RetryPolicy, ObservabilityConfig for deployment targets
- **Go SDK**: JSONSchema() methods on enum types for proper schema generation
- **Go SDK**: Typed platform config fields on Target (ClaudeCode, GeminiCLI, etc.)

### Changed

- **Breaking (Go SDK)**: `Target.Config json.RawMessage` replaced with typed config fields
  - Migration: Use `target.ClaudeCode = &ClaudeCodeConfig{...}` instead of `target.Config = json.Marshal(...)`
- **TypeScript SDK**: Now auto-generated from JSON Schema (was manually written)
- JSON Schemas regenerated with proper enum values and descriptions

### Removed

- **Go SDK**: `Target.Config` field (use typed config fields instead)
- **TypeScript SDK**: Manual `schemas.ts` (replaced by generated schemas)

## [v0.3.0] - 2026-01-24

### Highlights

- Add namespace support for organizing agents into subdirectories

### Added

- Optional `namespace` field in agent schema for organizing agents
- Recursive directory scanning in `LoadAgentsFromDir` - agents in subdirectories get namespace from directory name
- `LoadAgentsFromDirFlat` function for non-recursive loading (backward compatibility)
- `LoadAgentFromFile`, `LoadTeamFromFile`, `LoadDeploymentFromFile` loader functions in Go SDK
- `Namespace` field on Agent struct in Go SDK
- `AllowedTools` field on Agent struct in Go SDK
- `QualifiedName()` method on Agent to get `namespace/name` format
- `ParseQualifiedName()` function to split qualified names
- `WithNamespace()` builder method on Agent
- YAML struct tags on all Agent fields
- Support for `namespace/agent-name` references in team schema (agents, orchestrator, step.agent)

### Documentation

- README section on nested agent directories with examples

## [v0.2.0] - 2026-01-17
